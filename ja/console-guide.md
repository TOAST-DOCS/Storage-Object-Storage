<a id="storage-class"></a>

### ストレージクラス
データアクセス頻度、コスト要件に応じてストレージクラスを選択できます。頻繁にアクセスするデータのためのStandardクラスと、アクセス頻度が低いデータを安価な料金で長期保存できるEconomyクラスを提供します。

> [参考]
> すでに作成されたコンテナのストレージクラスは変更できません。
> Economyクラスコンテナにアップロードされたオブジェクトは最小保存期間30日が適用されます。30日以前に削除されたオブジェクトに対しても残余保存期間の料金が課金されます。
> EconomyクラスコンテナはAPIリクエスト1,000件あたり料金が適用されます。（HEAD/DELETEリクエスト除く）

<a id="set-object-lock-cycle"></a>
#### オブジェクトロック設定
オブジェクトロックコンテナにアップロードしたオブジェクトは**WORM（Write-Once-Read-Many）**モデルを使用して保存されます。オブジェクトロックコンテナにアップロードしたオブジェクトには、ロック満了日が設定されます。各オブジェクトに設定されたロック満了日以前には、オブジェクトを上書きしたり削除したりできません。

<a id="set-object-encryption"></a>
#### 暗号化設定
暗号化コンテナにアップロードするオブジェクトは、NHN CloudのSecure Key Managerサービスで管理する対称キーを使用して暗号化されます。したがって、暗号化コンテナを作成するには事前にSecure Key Managerサービスで対称キーを生成する必要があります。

暗号化コンテナに適用されるポリシーは次のとおりです。

* 暗号化コンテナにアップロードするオブジェクトは設定された対称キーを使用して暗号化されて保存されます。
* 暗号化されたオブジェクトをダウンロードすると復号化されて転送されます。
* 暗号化コンテナのオブジェクトをコピーまたはリージョン間コンテナレプリケーションを通じて他のコンテナにコピーすると、対象コンテナの暗号化設定に応じて再暗号化されるか復号化されて保存されます。
* 暗号化コンテナを作成する際に登録した対称キーIDは変更できません。対称キーを変更するには、Secure Key Managerサービスでキーローテーションを使用する必要があります。
* Secure Key Managerサービスで暗号化コンテナに設定した対称キーをローテーションした後、新しいオブジェクトをアップロードすると、以前のバージョンキーで暗号化されたオブジェクトがローテーションされたキーで再暗号化されます。この作業は使用量に応じて処理時間が長くなる場合があります。再暗号化が完了する前に以前のバージョンキーを削除しないよう注意が必要です。

> [注意]
Secure Key Managerサービスで暗号化コンテナに設定した対称キーを削除すると、暗号化されたオブジェクトを復号化できません。対称キーを誤って削除しないよう注意して管理する必要があります。

<a id="empty-a-container"></a>

### コンテナを空にする
選択したコンテナ内部のすべてのオブジェクトを削除します。

> [参考]
> ロック満了日が過ぎていないオブジェクトは削除されません。
> 選択したコンテナ内部のマルチパートオブジェクトはマニフェストオブジェクトのみ削除されます。他のコンテナに位置するセグメントオブジェクトは削除されません。

> [注意]
> レプリケーション設定を使用中の場合、対象コンテナのオブジェクトも一緒に削除される場合があります。
> コンテナを空にする作業を進行中のコンテナにオブジェクトをアップロードする場合、削除される可能性があります。

<a id="delete-container"></a>

### コンテナ削除
選択したコンテナを削除します。コンテナを削除する前にコンテナが空であることを確認する必要があります。コンテナの中にオブジェクトが残っている場合は削除されません。

コンテナのアクセスポリシーを**PUBLIC**に設定し、インデックスドキュメントとエラードキュメントを入力すると、コンテナで静的ウェブサイトをホスティングできます。静的ウェブサイトURLはコンテナリストで**URLコピー**ボタンをクリックして取得できます。

静的ウェブサイトのインデックスドキュメント、エラードキュメントとして使用するオブジェクトは、1つ以上の英数字または一部特殊文字（`-`、`_`、`.`、`/`）で構成された名前である必要があり、ファイル拡張子が`.html`のハイパーテキスト形式である必要があります。条件に合わない場合は設定できなかったり、静的ウェブサイトが動作しない可能性があります。

静的ウェブサイトのエラードキュメント名は`{レスポンスコード}{サフィックス}`の形式です。例えば、エラードキュメントを`error.html`に設定した場合、404エラーが発生した時に表示されるエラードキュメントの名前は`404error.html`になります。各エラー状況に応じてエラードキュメントをアップロードして使用できます。エラードキュメントを定義しなかったり、レスポンスコードに合うエラードキュメントオブジェクトがない場合は、ウェブブラウザのデフォルトエラードキュメントが表示されます。
<br/>

<a id="set-container-cors"></a>
##### Cross-Origin Resource Sharing（CORS）変更

ブラウザからObject Storage APIを直接呼び出すには、Cross-Origin Resource Sharing設定が必要です。Cross-Origin Resource Sharing項目の変更ボタンをクリックすると、許可する出典URLを登録できます。URLにはプロトコル（`https://`または`http://`）が含まれている必要があります。`*`を登録すると、すべての出典を許可できます。

<br/>

<a id="set-container-upload-policy"></a>
##### アップロードポリシー設定変更
コンテナにオブジェクト名ベースのアップロードポリシーを設定します。アップロードポリシー設定を活用すると、名前に特定の拡張子やキーワードが含まれたオブジェクトのみアップロードできるようにしたり、アップロードできないように制限したりできます。

アップロードポリシーは`ホワイトリスト`または`ブラックリスト`を設定でき、両方式を同時に設定することはできません。アップロードするファイルの拡張子またはファイル名に含まれるキーワードを設定できます。ただし、パスが含まれたオブジェクトは、パスを除いたオブジェクト名がポリシーに反映されます。アップロードポリシーは設定後から新規アップロードされるオブジェクトに適用されます。

ホワイトリストで拡張子`exe`、`jpg`を設定すると、`.exe`、`.jpg`拡張子を持つオブジェクトのみアップロードできます。ファイル名`example`を追加設定すると、`exe_example.exe`、`image_example.jpg`のように設定されたファイル名と拡張子を両方持つオブジェクトのみアップロードできます。


ブラックリストの場合、拡張子`exe`、`jpg`を設定すると、すべての`.exe`、`.jpg`拡張子を持つオブジェクトはアップロードできません。ファイル名`example`を追加設定すると、`test.exe`、`image.jpg`のような制限された拡張子のファイルと、`text_example.txt`のような制限されたキーワードを持つファイルの両方ともアップロードできません。
<br/>

<a id="set-container-replication"></a>
#### 複製

複製設定を通じて、コンテナのオブジェクトを他のリージョンのコンテナに複製できます。複製設定は災害復旧（disaster recovery）のための機能で、元のリージョンのオブジェクトを対象リージョンに同一に複製し、管理します。複製はバックグラウンドで一定周期ごとに進行されます。

複製ポリシーは次の通りです。

* **基本動作**
    * 元のコンテナでオブジェクトを変更（アップロード、メタデータ更新、削除）すると、複製対象コンテナに同一に反映されます。
    * 複製対象コンテナで変更されたオブジェクトは、元のコンテナに反映されません。
    * 複製はオブジェクトの最終変更時刻を基準に動作します。複製対象コンテナのオブジェクトが元のオブジェクトより後に変更された場合は、複製されません。
* **設定注意事項**
    * 複製対象コンテナとしては空のコンテナ使用を推奨します。対象コンテナに元のコンテナのオブジェクトと同じ名前のオブジェクトがすでにある場合、複製が円滑でない可能性があります。
    * 複製対象コンテナに同一名のオブジェクトが削除された履歴がある場合、複製されたオブジェクトの最近更新時間が複製設定時間に変更される可能性があります。
    * 元のコンテナと複製対象コンテナの名前を同一に設定することを推奨します。コンテナの名前が異なると、複製された大容量オブジェクトアクセスに失敗する可能性があります。
    * マルチパートアップロードした大容量オブジェクトのセグメントオブジェクトが他のコンテナに保存されている場合、セグメントが保存されたコンテナも同じ方式で複製設定を行う必要があり、複製された大容量オブジェクトにアクセスできます。
* **設定変更注意事項**
    * 複製設定を使用しないに変更すると、複製は中断されますが、すでに複製されたオブジェクトは維持されます。
    * 複製方向を転換すると、複製対象コンテナのオブジェクトが元のコンテナに復旧されます。元のコンテナで削除されたオブジェクトも復旧対象に含まれ、復旧されたオブジェクトの最近更新時間は複製設定時間に変更されます。
    * 複製対象コンテナを削除すると、同じ名前のコンテナを再び作成しても複製が再開されません。複製を再開するには、複製設定を再度行う必要があります。
* **制限及び例外事項**
    * 複製対象コンテナを他のリージョンに複製したり、他のコンテナの複製対象コンテナとして重複設定することはできません。
    * ライフサイクルが満了したがまだ削除されていないオブジェクトは、対象コンテナに複製される時にライフサイクル設定が除去されます。その後、元のコンテナで削除されると、削除内容が対象コンテナに伝播されて削除されます。
    * アーカイブコンテナの削除マーカーオブジェクトは複製されません。

> [注意]
> 暗号化コンテナを複製対象コンテナに指定した後、暗号化コンテナの対称キーをSecure Key Managerサービスで削除すると、暗号化コンテナ複製に失敗します。
<br/>

<a id="resume-container-replication"></a>
##### 複製開始

停止されたコンテナ複製を再開始します。複製は中断時点から続けて実行されます。
<br/>

<a id="suspend-container-replication"></a>
##### 複製停止

コンテナ複製を停止します。複製が停止された状態では、元のコンテナのオブジェクトを削除または修正しても複製されません。

> [注意]
> 複製停止期間中に削除された元のコンテナのオブジェクトは、対象コンテナに反映されない可能性があります。
<br/>

<a id="object"></a>

## オブジェクト

<a id="create-folder"></a>
### フォルダ作成
フォルダを作成します。フォルダはコンテナのオブジェクトをグループ化するための仮想的な単位です。WindowsのフォルダやLinuxのディレクトリと同様に、階層的にオブジェクトを管理できるように支援します。フォルダ名は英文256文字、ハングル85文字に制限されます。

> [参考]
> オブジェクトストレージのフォルダは、ファイルシステムが提供するディレクトリとは概念が異なります。便宜上提供する疑似（pseudo）フォルダです。フォルダを作成すると、実際には`{フォルダ名}/`という空のオブジェクトが作られます。フォルダ内のオブジェクトは`{フォルダ名}/{オブジェクト名}`形式の名前を持ちます。オブジェクトコピー機能を利用して新しいフォルダにオブジェクトをコピーすると、`{フォルダ名}/`形式の空のオブジェクトなしに直接`{フォルダ名}/{オブジェクト名}`形式のオブジェクトが作られます。したがって、このコピーされたオブジェクトを削除する場合、フォルダまで削除されたように見えます。事前に作成したフォルダにコピーする場合は、フォルダのオブジェクトを削除してもフォルダは残ります。

<a id="delete-folder"></a>
### フォルダ削除
フォルダを削除します。フォルダ内のすべてのオブジェクトとフォルダオブジェクトを削除します。
フォルダ内のマルチパートオブジェクトはマニフェストオブジェクトのみ削除され、選択項目に含まれていないセグメントオブジェクトは削除されません。

<a id="upload-object"></a>
### オブジェクトアップロード
すべてのオブジェクトはコンテナ内にアップロードする必要があります。オブジェクト1つの最大容量は5GBに制限されます。

> [参考]
> Webコンソールでは5GBを超えるファイルはアップロードできません。アップロードするオブジェクトの容量が5GBを超える場合は、`split`などのコマンドラインツールを使用して分割するか、ユーザーアプリケーションで5GB以下のサイズに分割してアップロードするようにプログラミングする必要があります。詳細な使用方法については、APIガイドの[マルチパートアップロード](/ja/Storage/Object%20Storage/ja/api-guide/#multipart-upload)を参照してください。

<a id="download-object"></a>
### オブジェクトダウンロード
選択したオブジェクトをダウンロードします。コンテナを作成する際にコンテナアクセスポリシーを**PRIVATE**に設定した場合は、許可されたユーザーのみオブジェクトにアクセスできます。**PUBLIC**に設定した場合は、リストで`URL コピー`ボタンをクリックしてオブジェクトの公開URLを確認できます。このURLでオブジェクトのハイパーリンクを作成したり、オブジェクトを直接ダウンロードしたりできます。

<a id="copy-or-move-object"></a>
### オブジェクトコピー/移動
オブジェクトを指定したコンテナにコピーまたは移動します。複数のオブジェクトを選択して他のコンテナまたは同じコンテナの新しいパスにコピーまたは移動できます。

> [参考]
> 入力できるパスの最大長は、オブジェクト名によって可変的です。コピー先のパスにオブジェクト名を加えた長さが1024バイト以下である必要があります。
> `{パス最大長} = 1024 - {オブジェクト名長} - 1`
>
> マルチパートオブジェクトはマニフェストオブジェクトのみコピーまたは移動されます。

<a id="delete-object"></a>
### オブジェクト削除
選択したオブジェクトを削除します。複数のオブジェクトを同時に選択して削除できます。

> [参考]
> マルチパートオブジェクトを削除すると、選択したマニフェストオブジェクトのみ削除されます。選択していないセグメントオブジェクトは削除されません。

<a id="create-signed-url"></a>
### 署名付きURL生成
ロールベースアクセスポリシーに関係なく、設定した時間中に指定したオブジェクトに自由にアクセスできる署名付きURLを生成します。

> [参考]
> 単一オブジェクトのみ選択でき、フォルダオブジェクトは選択できません。
> 有効期間は最大720分まで分単位で設定できます。

> [注意]
> 署名付きURLが流出すると、誰でも選択したオブジェクトにアクセスできるため、注意して使用する必要があります。署名付きURLが流出しても被害を軽減できるよう、状況と用途に合わせて適切な有効期間を設定して使用することを推奨します。

<a id="manage-object"></a>
### オブジェクト管理
選択したオブジェクトの基本情報を確認し、属性を管理します。

> [参考]
> オブジェクト有効期限日とロック有効期限日の両方を設定する場合、オブジェクト有効期限日は常にロック有効期限日以降に設定する必要があります。

<a id="set-object-expiration"></a>
#### オブジェクト有効期限日変更

選択したオブジェクトの有効期限日を変更できます。

<a id="set-object-lock-expiration"></a>
#### オブジェクトロック日変更

選択したオブジェクトのロック有効期限日を変更できます。以前に設定された有効期限日より前には変更できません。

<a id="prefix-search"></a>

## 接頭語検索
検索窓に接頭語を入力して**検索**ボタンをクリックすると、入力した接頭語で始まるコンテナ、フォルダ、オブジェクトを検索できます。コンテナリストではコンテナを、オブジェクトリストではフォルダとオブジェクトを検索します。

<a id="s3-api-credentials"></a>

## S3 API認証情報
Amazon S3互換APIを使用するための認証情報を発行できます。S3 API認証情報は有効期限がなく、ユーザー別にプロジェクト当たり最大3個まで発行できます。

> [注意]
> S3 API認証情報キーが流出すると、誰でも流出したキーを利用してオブジェクトにアクセスできます。キーが流出した場合は、流出した認証情報を削除して新しく発行して使用することを推奨します。
