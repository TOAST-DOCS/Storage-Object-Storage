## 서드 파티 도구 가이드

이 문서는 서드 파티 도구를 통해 NHN Cloud의 오브젝트 스토리지 서비스를 사용하는 방법을 설명합니다.

## Cyberduck

Cyberduck은 오픈소스 서버 / 클라우드 스토리지 브라우저입니다. Cyberduck을 통해 오브젝트 스토리지을 편리하게 관리할 수 있습니다.

### Cyberduck 프로그램 다운로드

[Cyberduck 다운로드 페이지](https://cyberduck.io/download/)를 통해서 본인의 운영 체제에 맞는 파일을 다운로드합니다.

### 오브젝트 스토리지 연결 설정

#### Cyberduck 프로필 다운로드 / 책갈피 설정

Cyberduck에 사용할 프로필을 [다운로드]()한 뒤, 실행하여 새로운 책갈피를 생성하고 추가 내용을 입력합니다.

| 항목                  | 설명                                                                                                                                                                              |
|---------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 별명                  | 책갈피의 별칭입니다. 원하는 값으로 자유롭게 설정할 수 있습니다.                                                                                                                                            |
| Project_Id:Email_Id | Project_Id : 사용자의 Project ID 입니다. <br/> 콘솔 상단 프로젝트 우측의 톱니바퀴 클릭 > **프로젝트 기본 정보** > **프로젝트 ID** 에서 확인할 수 있습니다.<br/><br/> Email_Id : NHN Cloud 사용자는 Email 주소를, IAM 사용자는 ID를 입력합니다. |
| API Password        | 오브젝트 스토리지 서비스 페이지의 **API Endpoint 설정** 버튼을 클릭해 설정한 비밀번호를 입력합니다.                                                                                                                 |

> [참고]
API 비밀번호 설정 방법은 [오브젝트 스토리지 API 가이드](api-guide/#_1) 항목을 참조하세요.

#### 오브젝트 스토리지 연결 방법

설정한 책갈피를 더블클릭하여 오브젝트 스토리지에 접속합니다.

### 컨테이너 / 오브젝트 조회

오브젝트 스토리지에 접속하면 **모든 리전**의 컨테이너 리스트가 브라우저에 나타납니다.

원하는 컨테이너를 더블클릭하여 컨테이너 내 오브젝트를 조회할 수 있습니다.

> [참고]
> 각기 다른 리전에 같은 이름의 컨테이너가 있을 때에도 Cyberduck에서는 동일한 이름으로 나타납니다.
> 메뉴의 **보기** > **열** > **지역** 을 선택해서 열 항목에 리전을 표시할 수 있습니다.

### 컨테이너 생성

컨테이너 리스트의 빈 공간을 마우스 오른쪽 버튼 클릭 > `새 폴더...` 를 통해서 새로운 컨테이너를 생성합니다.

생성 요청 후 컨테이너 리스트의 빈 공간을 마우스 오른쪽 버튼 클릭 > `다시보기` 를 진행하여 리스트를 업데이트할 수 있습니다.

> [참고]
> 컨테이너 생성 시 리전을 선택할 수 있습니다.

### 오브젝트 업로드

컨테이너를 선택한 뒤 메뉴의 `동작` > `업로드...` 를 클릭하거나 마우스 오른쪽 버튼 클릭 > `업로드...` 를 클릭하여 오브젝트를 선택하고 업로드합니다.

또한 끌어다 놓기로 파일을 업로드할 수 있습니다.

> [참고]
>
> Cyberduck으로 폴더를 업로드하면 폴더 이름과 동일한 0바이트 오브젝트가 하나 더 생성됩니다. 이는 콘솔에서만 확인되며, 해당 오브젝트는 제거해도 무관합니다.

### 오브젝트 다운로드

오브젝트를 선택한 후 마우스 오른쪽 버튼 클릭 > `내려받기` 를 클릭하여 다운로드합니다.

또한 끌어다 놓기로 오브젝트를 다운로드할 수 있습니다.


> [참고]
>
> 오브젝트를 다운로드하면 기본적으로 로컬의 **다운로드** 폴더로 저장됩니다. 특정 위치로 저장을 원하면 마우스 오른쪽 버튼 클릭 > **지정된 위치로 내려받기** 를 통해서 진행할 수 있습니다.
> 오브젝트를 업로드 / 다운로드하면 **전송** 창이 팝업되어 진행 상태를 파악할 수 있습니다.

### 오브젝트 삭제

삭제할 오브젝트를 마우스 오른쪽 버튼 클릭 > `삭제`를 클릭하여 삭제합니다.

> [주의]
> 컨테이너 내부에 오브젝트가 있어도 전부 삭제됩니다.

### 폴더 동기화

폴더를 선택하고 마우스 오른쪽 버튼 클릭 > `동기화`를 통해 로컬의 폴더와 오브젝트 스토리지 간의 동기화를 진행합니다.

다운로드 / 업로드 / 미러 3가지로 설정할 수 있습니다.

#### 다운로드

오브젝트 스토리지의 오브젝트들을 아래와 같은 기준으로 검사한 후, 해당하는 오브젝트를 로컬 폴더에 다운로드합니다.

- 로컬 폴더에 없는 오브젝트
- 체크섬이 다른 오브젝트
- 오브젝트 스토리지의 타임스탬프가 신규인 오브젝트

#### 업로드

로컬 폴더의 파일들을 아래와 같은 기준으로 검사한 후, 파일을 오브젝트 스토리지에 업로드합니다.

- 오브젝트 스토리지 컨테이너에 없는 파일
- 체크섬이 서로 다른 파일
- 로컬 폴더의 타임스탬프가 신규인 파일

#### 미러

파일들은 로컬과 오브젝트 스토리지 양방향으로 전달되며, 타임스탬프를 기반으로 없거나 변경된 파일을 양쪽 시스템에 서로 생성하거나 변경합니다.


## SwiftStack

SwiftStack은 오브젝트 스토리지 플랫폼입니다. SwiftStack을 통해 오브젝트 스토리지를 편리하게 관리할 수 있습니다.

### SwiftStack 프로그램 다운로드

[SwiftStack 다운로드 페이지](https://platform.swiftstack.com/docs/install/swiftstack_client.html)에서 본인의 운영 체제에 맞는 클라이언트를 다운로드 할 수 있습니다.

### 오브젝트 스토리지 연결 설정

#### 어카운트 생성

브라우저 좌측의 `Accounts` > `Add account` 를 통해서 어카운트를 생성합니다. 생성에 필요한 정보는 아래와 같습니다.

| 항목           | 설명                                                                                                                                                                             |
|--------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Auth Type    | **V2**를 선택합니다.                                                                                                                                                                 |
| Username     | NHN Cloud 사용자 Email 혹은 IAM 사용자 ID를 입력합니다.                                                                                                                                      |
| Password     | 오브젝트 스토리지 서비스 페이지의 **API Endpoint 설정** 버튼을 클릭해 설정한 비밀번호를 입력합니다.                                                                                                                |
| Auth URL     | 오브젝트 스토리지 서비스 페이지의 **API Endpoint 설정** 버튼을 클릭하여 확인할 수 있습니다.<br/> 콘솔 > **Storage** > **Object Storage** > **API 엔드포인트 설정** > **신원 서비스(identity)** 의 주소 끝에 '/tokens'를 붙여서 입력합니다. |
| Project Name | 사용자의 Project ID 입니다. <br/> 콘솔 상단 프로젝트 우측의 톱니바퀴 클릭 > **프로젝트 기본 정보** > **프로젝트 ID**에서 확인할 수 있습니다.                                                                                 |    
| Region       | 사용할 리전을 지정할 수 있습니다.                                                                                                                                                            |

> [참고]
>
> API 비밀번호 설정 방법은 [오브젝트 스토리지 API 가이드](api-guide/#_1) 항목을 참조하세요.
> 입력값이 올바르지 않으면 Region을 지정할 때 오류 팝업창이 나타납니다.

입력값을 올바르게 입력한 뒤, 하단의 `Save`을 클릭해 새로운 어카운트를 생성합니다.

#### 오브젝트 스토리지 연결 방법

생성한 어카운트를 더블클릭하여 오브젝트 스토리지에 접속합니다.

### 컨테이너 / 오브젝트 조회

오브젝트 스토리지에 접속하면 설정한 리전의 컨테이너 리스트가 브라우저에 나타납니다.

원하는 컨테이너를 더블클릭하여 컨테이너 내 오브젝트를 조회할 수 있습니다.

### 컨테이너 생성

브라우저 상단의 `Add Container` > `Edit Before Creation` 을 통해서 생성할 컨테이너의 이름을 지정합니다.

이름을 입력한 후 하단의 `Create`을 클릭해서 컨테이너 생성을 요청합니다.

> [참고]
> 생성 요청 후 브라우저 상단의 `Refresh` 를 진행하여 리스트를 업데이트할 수 있습니다.

### 오브젝트 업로드

브라우저 상단의 `Upload` > `Choose File(s)` 버튼을 클릭하여 업로드할 오브젝트를 선택하고 업로드합니다.

또한 끌어다 놓기로 파일을 업로드할 수 있습니다.

> [참고]
>
> `Upload` 버튼을 통해서 2개 이상의 파일을 업로드하면 하단의 `Object Name Prefix`을 통해 업로드할 파일들 이름 앞에 접두사를 붙일 수 있습니다.

### 오브젝트 다운로드

#### 폴더 다운로드

폴더 리스트 우측의 `Actions`에 있는 다운로드 버튼을 통해서 원하는 폴더를 다운로드힙니다.

또한, 원하는 폴더들을 리스트 좌측 체크박스에서 체크한 뒤, 브라우저 상단의 `Selection` > `Download` 를 통해서 다운로드합니다.

#### 파일 다운로드

원하는 파일들을 리스트 좌측 체크박스에서 체크한 뒤, 브라우저 상단의 `Selection` > `Download` 를 통해서 다운로드합니다.

### 삭제

파일 / 폴더 리스트 우측의 `Actions`에 있는 삭제 버튼을 통해서 원하는 오브젝트를 삭제합니다.

또한 원하는 오브젝트들을 리스트 좌측 체크박스에서 체크한 뒤, 브라우저 상단의 `Selection` > `Delete` 를 통해서 삭제할 수 있습니다.

> [주의]
>
> 컨테이너 내부에 파일이 있어도 전부 삭제됩니다.
